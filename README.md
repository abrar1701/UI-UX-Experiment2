## ðŸ§  Application of Gestalt Principles (Requirement A)

1.  **Principle of Proximity**
    * *Application:* UI elements that belong together (e.g., icon, title, description in a card) are grouped closely, while the card itself is separated from others by whitespace.
2.  **Principle of Similarity**
    * *Application:* Interactive elements share styles. All primary CTA buttons use the Primary Color; all text links are colored/underlined similarly.
3.  **Principle of Common Region**
    * *Application:* Distinct background colors group elements. The Header and Footer have different shades than the main content to separate page structure.
4.  **Principle of Closure**
    * *Application:* Cards use simple borders or shadows. The user perceives the elements within as a single unit, even without heavy boxing.

## ðŸ‘† Interaction Behaviors and Usability (Requirement C)

### Interaction Behaviors
* **Hover States (Web):** Links/buttons change color or underline on mouse-over. Cards have a subtle "lift" (box-shadow).
* **Active/Pressed States:** Buttons darken when pressed. Current page in nav is highlighted.
* **Feedback:** Forms use green outlines for valid input and red for errors. Loading spinners appear upon submission.

### Usability Principles (Nielsen's Heuristics)
* **Visibility of System Status:** Users are informed via active nav links, spinners, and success/error messages.
* **Consistency and Standards:** Follows conventions (Logo top-left, Hamburger menu on mobile).
* **Aesthetic and Minimalist Design:** Generous whitespace and clutter-free layout to reduce cognitive load.
* **Flexibility and Efficiency:** Mouse users get horizontal nav; touch users get bottom nav/stacked content.
